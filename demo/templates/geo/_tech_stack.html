Tech Stack

How It Works

This page demonstrates using pure Django and Django-Rest-Framework as backend and DataTables and Django Templates as base for frontend on one page.

Data is initialized through feature files.

Tech Stack

  • Django
  • Django Rest Framework
  • PostgreSQL
  • DataTables

This page uses Django to get HTML content which is inherited from base template with menus, breadcrumbs, etc., and includes table based on DataTable plugin, as well as javascript and css styles.

Table provides visualizes Database records from City model (db table), so Django models make sense here.

As well as Django Template Language, which is used to generate html page using inheritance, "include", "extend", "url", "static" template tags and context processors.

Django application is ran under Daphne as a separate service.

DRF handles API requests and does it well.

It allows to create API views using various number of base classes and simplifies developer work.

See DRF Swagger page for details.

PostgreSQL is used as a database for Django.

We may use any other RDBM like MySQL, MSSQL, Sqlite.

Datatables allows to visualize well structured data and bring such functionality like paging, ordering, filtering into tables.

See https://datatables.net/ for details.
